# 팜바톤 (FarmBaton)

고령 농가의 농장(농지+작목+시설+판로)을 청년농에게 잇는 승계 진단·매칭 플랫폼.
제11회 농업·농촌 공공데이터+AI 활용 창업경진대회 출품작. **접수 마감 2026-06-30.**

## 스코프 (변경 금지)

- 지역: 충북·경북·충남 3개 도
- 작목: 사과·복숭아·포도 (과수 3종만)
- MVP 데모 기능 2개: ① 농가 등록→인수 검토 리포트 ② 청년농 프로필→매칭 리스트
- RAG 챗봇은 7월 작업. 6월 중 절대 착수하지 않는다.

## 아키텍처

- backend: FastAPI + PostgreSQL/PostGIS (Supabase)
- frontend: React(Vite) + Leaflet, 모바일 퍼스트 반응형
- 외부 API: 팜맵(data.go.kr B552895), V-World(지오코딩·공시지가·배경지도), KAMIS
- 정적 데이터: 실거래가 CSV(rt.molit.go.kr), 농산물소득조사, 팜맵 SHP, 중개업소 현황

## 디렉터리

```
farmbaton/
  backend/app/        # FastAPI (routers, services, models)
  backend/tests/      # pytest
  frontend/src/
  etl/                # 1회성 적재 스크립트 (SHP, CSV)
  db/seed/            # facility_std.csv, orchard_age_coef.csv, seed_reference_tables.sql
  docs/formula.md     # 가치평가 산식 확정본 (단일 진실 원천)
```

## 핵심 산식 (docs/formula.md 요약)

- 예상 연소득 = 10a당 평균소득(income_coef) × 면적(10a) × 수령계수(orchard_age_coef) × 시세보정(KAMIS)
- 시설 잔존가 = 표준단가(facility_std) × 면적 × max(잔존가율, 1−경과연수/내용연수) × 상태등급(facility_condition)
- 인수 검토가 범위 = 토지 기준가(공시지가×면적, 실거래 보정) + 시설 잔존가 + 영업권(자료 제출 시 순소득×1~2배) → 반드시 min/max 범위로 산출

## 절대 규칙

1. **수치 계산에 LLM 호출 금지.** 가치평가·매칭 점수는 전부 결정론적 Python 함수. LLM(Claude API)은 설명문 생성에만 사용.
2. 화면·문서 표기는 "감정가" 금지 → **"인수 검토가 범위(참고용 추정)"** + 면책 문구 필수.
3. 모든 외부 API 호출에 정적 폴백 구현 (데모가 절대 죽지 않게). 실거래가는 API 호출 금지, DB 적재본만 사용.
4. 공간 데이터는 DB에 EPSG:4326으로 통일 저장 (SHP 원본 5186 → 적재 시 변환).
5. 단가·계수 등 기준값은 코드에 하드코딩 금지, 반드시 DB 테이블 참조.
6. 시드 농장 데이터는 실존 필지 기반이되 소유주는 가상 인물로 명시 (is_demo=true 플래그).
7. .env의 키(Supabase, V-World, KAMIS)는 커밋 금지. .env.example만 갱신.
8. 데이터·키·파일이 없거나 형식이 예상과 다르면 절대 임의 가정·더미 생성으로
   진행하지 않는다. 작업을 멈추고 [데이터 요청] 블록으로 정훈에게 묻는다.
   형식:
   [데이터 요청]
   - 무엇이: (필요한 파일/키/값)
   - 어디서: (발급·다운로드 경로)
   - 왜: (이게 없으면 막히는 작업)
   - 임시 대안: (있다면. 없으면 "없음 — 대기")
9. 검증(SELECT COUNT, 샘플 출력, 테스트)을 통과하기 전에는 다음 단계로
   넘어가지 않고, 결과를 보고한 뒤 승인을 기다린다.
   
## 작업 방식

- 작업 시작 전 계획을 먼저 제시하고 승인받은 뒤 구현한다.
- 한 작업 = 한 커밋. 커밋 메시지는 `[P1] shp 적재 스크립트` 형식.
- 가치평가 엔진은 테스트 우선: docs/formula.md의 예시 케이스를 pytest로 먼저 작성 후 구현.
- 구현 후 실행 검증까지 마치고 결과(테스트 출력, curl 응답)를 보고한다.
- 스코프에 없는 기능 제안 금지. 발견한 개선점은 docs/backlog.md에 기록만.
