- **왜**: 실제 필지는 혼식(한 농장에 작목 2종 이상)인 경우가 흔함. 다만 가치평가 엔진(`backend/app/services/valuation.py:calc_total_value`)과 매칭 점수(`crop_score`)가 전부 "농장당 crop_code 1개"를 전제로 짜여 있어서 지원하려면 소득 계산을 작목별 가중합산으로 다시 설계해야 함 — 핵심 산식·스키마 변경이라 영향 범위가 큼.
- **참고**: 팜맵 필지 자체가 품종 정보를 안 담고 있어서(영농형태 코드만 "과수원"), 이 기능이 있어도 "이 필지에 실제로 어떤 작목이 섞여 있는지"는 여전히 알 수 없음 — 시드 데이터 라벨링 문제의 해결책은 아니고, 실제 농가가 직접 등록할 때를 위한 기능.
