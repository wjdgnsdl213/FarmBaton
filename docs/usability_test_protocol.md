# 팜바톤 사용성 미니테스트 진행 가이드

> 목적: 2차 서면검토 지적 ①(실사용자 검증 부재, 고령·청년 모바일 사용성) 대응.
> 발표(7/29~30)에서 "지적받고 즉시 검증했다"는 수치 확보.
> 근거: 사용성 테스트는 5명이면 주요 문제의 약 80%를 발견한다(Nielsen, 1993).

## 1. 준비

- **참가자**: 고령층 3~5명(60대 이상, 스마트폰 사용자 — 부모님·친척·이웃) + 청년 3~5명(20~30대 지인)
- **장비**: 참가자 본인 스마트폰(중요 — 익숙한 기기여야 실제 사용성이 나옴), 타이머, 기록지(아래 양식 인쇄 또는 별도 폰에 띄우기)
- **주소**: farmbaton.vercel.app
- **소요**: 1인당 15~20분
- **사전 확인**: 테스트 전날 본인이 전체 플로우를 한 번 돌려 서버·로그인 정상 확인
- **동의**: "서비스 개선용 테스트이고, 결과는 익명으로 기록됩니다" 구두 동의.
  화면 녹화 가능하면 최고, 어려우면 옆에서 메모.
- **데이터 정리**: 테스트 중 만든 계정·농장은 목록을 적어뒀다가 테스트 후 삭제 요청

## 2. 진행 규칙 (가장 중요)

1. **절대 먼저 시연하거나 설명하지 않는다.** 태스크만 주고 지켜본다.
2. 참가자가 물어보면 → "어떻게 하실 것 같으세요?"로 되묻는다.
3. 막혀도 **30초는 침묵**하고 기다린다.
4. **2분 넘게 못 넘어가면 힌트 1회** 주고 '부분성공'으로 기록, 다음 태스크로.
5. 참가자가 소리 내며 생각하도록 유도: "지금 무슨 생각 하시는지 말씀해주세요."
6. 실패는 참가자 잘못이 아니라 화면 잘못이다 — "어렵네요" 하면 "저희가 잘못 만든 겁니다" 라고 답한다.
7. **막힌 화면은 즉시 스크린샷** (수정 전 증거 = 발표 before/after 재료).

## 3. 태스크 — 고령층 (농장주 플로우)

| # | 태스크 (읽어줄 문장) | 성공 기준 |
|---|---|---|
| T1 | (첫 화면 5초 보여주고 가리기) "무슨 서비스 같으세요?" | 승계/농장/매칭 중 1개 언급 |
| T2 | "회원가입하고 로그인해보세요" | 도움 없이 로그인 상태 도달 |
| T3 | "알고 계신 과수원 주소로 농장을 등록해보세요" (주소 하나 미리 준비) | 등록 완료 화면 도달 |
| T4 | "이 농장을 넘긴다면 얼마 정도 받을 수 있다고 나오나요?" | 인수 검토가 범위를 찾아 읽음 |
| T5 | "그 금액이 확정 가격일까요, 참고용일까요?" | '참고용/추정'이라고 답함 |

- T5가 특히 중요: "감정가 아님 + 범위 제시" 설계가 실사용자에게 실제로 전달되는지 확인
  → 발표 Q&A에서 "면책이 실제로 인지되는지 확인했다"고 답할 수 있는 근거.

## 4. 태스크 — 청년 (청년농 플로우)

| # | 태스크 | 성공 기준 |
|---|---|---|
| T1 | (첫 화면 5초) "무슨 서비스 같으세요?" | 승계/농장/매칭 중 1개 언급 |
| T2 | "귀농한다고 상상하고, 희망 조건을 등록해 농장을 추천받아보세요" | 매칭 리스트 도달 |
| T3 | "가장 마음에 드는 농장을 하나 골라보세요. 왜 그 농장인가요?" | 점수 외 근거(지역·가격 등) 1개 언급 |
| T4 | "이 매칭 점수는 뭘 의미하는 것 같나요?" | 조건 적합도 취지로 답함 |
| T5 | "그 농장을 인수하려면 얼마가 필요한지 찾아보세요" | 인수 검토가 범위 도달 |

## 5. 마무리 3질문 (전원 공통)

1. "가장 어려웠던 부분은 어디였나요?"
2. "주변 분께 이 서비스를 권하시겠어요? 왜요?"
3. "딱 하나만 고친다면 뭘 고칠까요?"

## 7. 테스트 후 — 결과 활용 흐름

1. 기록지 전체를 Claude에게 전달 (사진으로 찍어 붙여도 됨)
2. Claude가: 태스크별 성공률·평균 시간 집계 → 상위 이슈 1~2건 수정 → before/after 캡처 → 슬라이드 문구 생성
3. 슬라이드 14(기술성과→검증) 최종 형태:
   - pytest 55개 통과
   - 실거래 20만건 백테스트 — 과수원 중앙오차 18.5% (개선 24%→18.5%)
   - 사용자 테스트 N명 — 핵심 태스크 성공률 X%, 발견 이슈 Y건 중 Z건 즉시 개선
