## 4. 서비스 개요 *(300자 이내)*

> 팜바톤은 고령화로 멈추는 과수 농장을 청년농에게 잇는 농장 승계 진단·매칭 플랫폼입니다. 농장주가 주소만 입력하면 팜맵·개별공시지가·토지 실거래가·KAMIS 시세 등 공공데이터를 결합해 '인수 검토가 범위(참고용 추정)'와 예상 연소득을 자동 산출하고, 생성형 AI가 이를 농장주·청년농 관점별 리포트로 알기 쉽게 설명합니다. 청년농은 희망 지역·작목·자본 조건으로 매칭 점수가 높은 승계 가능 농장을 추천받아 바로 상담·대화할 수 있습니다. 정보 비대칭으로 사라지던 영농 기반을 데이터로 다음 세대에 잇습니다.

## 5. 개발 서비스의 기능 및 특징 *(평가: 기술성 20)*

### 5-1. 핵심 기능 2가지

**① 농가 등록 → 인수 검토 리포트 (농장주)**
- 주소 입력 → V-World 지오코딩으로 좌표 변환, **팜맵에서 필지 경계·면적 자동 취득**(직접 입력 불필요)
- 작목(사과·복숭아·포도)·수령·승계 방식 입력 → **결정론적 가치평가 엔진**이 즉시 산출
  - 추가로 **시설·판로·매출은 선택 입력**(보유 시설 목록·연 매출·주 판로) → 시설 잔존가·영업권까지 반영되고 **신뢰등급이 C→B로 상승**(자료를 낼수록 정밀해지는 단계적 평가)
  - 예상 연소득 = 10a당 평균소득(농축산물 소득조사) × 면적 × 수령계수 × 시세보정(KAMIS)
  - 토지 기준가 = 개별공시지가 × 면적, 토지 실거래가로 보정
  - 시설 잔존가 = 표준단가(시설 기준표) × 면적 × 잔존가율 × 상태등급
  - 영업권 = 매출·판로 자료 제출 시 순소득 기반으로 보수적 산정(자료 없으면 0)
  - → **인수 검토가 범위(참고용 추정)** 를 min~max 범위 + 신뢰등급(A~D)으로 제시
- **생성형 AI 리포트(PDF)**: 같은 수치를 농장주용(매도 준비·시설 가치 어필)과 청년농용(자본 회수·정책자금·영농 체크리스트) **관점별로 차별 서술**

**② 청년농 프로필 → 매칭 리스트 (청년농)**
- 희망 지역·작목·보유 자본·영농 경력·정책자금·승계 선호 입력
- **다요인 매칭 스코어링**(지역·작목·자본·경험·승계·정책 6개 항목, 100점 만점)으로 승계 가능 농장을 점수순 추천
- 농장별 시설 현황·맞춤 지원사업·PDF 리포트 확인 → **인앱 상담 신청 → 농장주 수락 시 채팅**으로 승계 논의 시작(양방향 대화)

### 5-2. 기술 아키텍처
- **백엔드**: FastAPI + PostgreSQL/PostGIS(Supabase) — 공간데이터 EPSG:4326 통일 저장
- **프런트엔드**: React(Vite) + Leaflet 지도, 모바일 퍼스트 반응형
- **공간정보 처리**: 팜맵 SHP(EPSG:5186) → 적재 시 4326 변환, PostGIS 공간연산
- **신뢰성 설계(절대 죽지 않는 데모)**: 모든 외부 API 호출에 정적 폴백 구현. 특히 V-World가 해외 클라우드 IP를 차단(공간정보 국외반출 제한)하는 문제를, **국내 회선에 띄운 자체 우회 프록시**(Tailscale Funnel)로 해결 → 임의 주소 실시간 지오코딩 가능
- **재현 가능한 계산**: 단가·계수 등 모든 기준값을 코드 하드코딩이 아닌 DB 기준테이블로 관리

### 5-3. 핵심 특징
- **결정론(숫자) + 생성형 AI(설명) 분리** — 금전 추정치는 재현 가능한 산식으로, 이해·의사소통은 AI로 (아래 6·7항)
- "감정가"가 아닌 **"인수 검토가 범위(참고용 추정)"** + 면책 문구로 책임 있는 표현
- 실존 필지 기반 시드 데이터(소유주는 가상 인물, `is_demo` 표기)로 현실감 있는 데모

## 6. AI 기술 활용 여부 *(평가: 공공데이터·AI 활용 30 — 최고 배점)*

### 주(主) AI 기술: 생성형 AI(LLM) 기반 개인화 리포트·설명 생성
- 가치평가 엔진이 산출한 다수의 공공데이터 기반 수치(예상 소득·토지·시설·영업권·신뢰등급)와 농장 특성을 입력받아, **읽는 사람의 입장(농장주 매도 vs 청년농 인수)에 맞춰 자연어 리포트를 생성**(Anthropic Claude API).

### "왜 이 AI가 꼭 필요한가" (AI 필연성)
1. **이해 격차 해소**: 승계의 핵심 당사자는 디지털·금융 정보에 익숙하지 않은 **고령 농가**와, 영농·자산 평가 경험이 부족한 **청년농**. 표·숫자만으로는 의사결정이 어렵다. 생성형 AI는 동일 데이터를 **각자의 언어와 관점으로 번역**해 정보 비대칭을 줄인다.
2. **1:1 맞춤의 확장성**: 농장마다·관점마다 다른 설명을 사람이 일일이 쓰는 것은 불가능. 생성형 AI만이 **농장 수 × 관점 수**의 개인화 서술을 즉시·저비용으로 제공한다.
3. **공공데이터와의 결합 필연성**: AI는 빈 말이 아니라 **팜맵·공시지가·KAMIS·소득조사 등 공공데이터로 산출된 실제 수치를 근거로** 서술한다. 공공데이터가 없으면 설명할 내용 자체가 없다.

### 책임 있는 AI 설계 (독창적 기술 판단)
- **금전 추정치 계산에는 LLM을 일절 사용하지 않는다.** 가치평가·매칭 점수는 100% **결정론적 Python 함수**로 산출 → 동일 입력에 동일 결과, 추적·검증 가능, **환각(hallucination)으로 인한 잘못된 금액 제시를 원천 차단**.
- LLM은 **설명문 생성에만** 사용. "숫자는 알고리즘, 설명은 AI"라는 역할 분리가 금융·자산 도메인에서 신뢰를 확보하는 핵심 설계.

### 보조 AI/데이터 지능
- **의사결정 지원**: 청년농↔농장 매칭은 공공데이터 다중 융합 기반 6요인 스코어링으로 "가장 잘 맞는 승계처"를 정량 추천(추천·의사결정 지원).

## 7. 기존 서비스와의 차별성 및 독창성 *(평가: 독창성 20)*

| 구분 | 기존 서비스 | 팜바톤 |
|------|-------------|--------|
| 대상 | 부동산 거래(매물 중개), 일반 농지은행 | **농장 "승계"** — 농지+작목+시설+판로를 묶은 영농 기반 전체 |
| 가치 산정 | 호가/감정평가(비용·시간 큼) | 공공데이터 결합 **인수 검토가 범위(참고용 추정)** 즉시·무료 |
| 정보 제공 | 정적 매물 정보 | **관점별(농장주/청년농) AI 리포트** + 매칭 점수 + 지원사업 |
| 연결 | 전화/오프라인 | **인앱 매칭·상담·채팅** 양방향 |

**독창적인 점**
1. **승계 특화 가치평가 모델** — 토지뿐 아니라 과수 수령계수·시설 잔존가·영업권까지 반영한 영농 자산 통합 추정(국내 유사 서비스 부재).
2. **결정론 계산 + 생성형 AI 설명의 분리 아키텍처** — 신뢰성과 개인화를 동시 달성.
3. **공공데이터 국외반출 제약을 우회하는 국내 프록시 인프라** — 해외 클라우드 배포에서도 V-World 공간정보를 안정적으로 활용하는 실전 엔지니어링.
4. **양면 매칭 + 정책자금 연계** — 청년창업농 정책자금 한도 내 인수 가능 농장을 우대 추천.

## 8. 제품 및 서비스 사업성 *(평가: 발전 가능성 20)*

### 8-1. 팀 구성 및 창업 추진 의지
**상호 보완적 3인 팀** (개발 + 디자인 + 농업 도메인)

| 팀원 | 전공 | 역할 |
|------|------|------|
| **김정훈** (대표) | 컴퓨터공학 | 기획·풀스택 개발 총괄 — 공공데이터 적재, 가치평가·매칭 엔진, 배포 전반 |
| **윤채원** | 컴퓨터공학 | UI/UX 디자인·사용성 검토 |
| **유수민** | 농업 계열 | 농업 도메인 자문·아이디어 — 승계 현장 이해, 작목·시설 검증 |

- **창업 유형·시점**: 예비창업자(개인·팀, 현재 사업자 미등록). 본 대회를 통한 MVP 검증을 거쳐 **2026년 하반기 개인 사업자등록**을 목표로 하며, 사업화 성과에 따라 법인 전환을 추진할 계획.
- **추진 의지**: 충북·경북·충남 과수 3작목으로 검증된 MVP를 전국·전작목으로 단계 확장. 지자체 농업기술센터·농지은행·청년농 지원사업과 연계한 B2G/B2B2C 모델로 사업화 추진.

### 8-2. 수익화 모델(안)
1. **승계 성사 중개 수수료**(매칭→상담→계약 전환 시)
2. **프리미엄 리포트/실사 연계**(감정평가사·공인중개사 연계 수수료)
3. **지자체·공공기관 구독형 대시보드**(지역 승계 사각지대 모니터링)
4. **정책자금·보험·영농 자재 제휴**

### 8-3. 시연 시나리오 (시제품 완료)
1. 농장주 로그인 → 주소 입력 → 팜맵 면적 자동 취득 → (선택) 시설·판로·매출 입력으로 영업권·시설 잔존가 반영·신뢰등급 상승 → 인수 검토 리포트(범위+신뢰등급) → 관점별 PDF 다운로드 → 매칭 풀 공개
2. 청년농 로그인 → 희망 조건 입력 → 매칭 점수순 농장 리스트 → 농장 상세·지원사업 → 상담 신청
3. 농장주 상담 수락 → 인앱 채팅으로 승계 논의
- ※ 모든 단계가 production(https://farmbaton.vercel.app)에서 실제 동작.

### 활용의 지속성·확장성
- **지속성**: 위 데이터는 모두 정기 갱신되는 상시 개방 데이터. 가격·시세는 API, 대용량/제약 데이터(실거래가)는 정적 적재본으로 안정 확보.
- **가공·확장**: 팜맵 SHP를 PostGIS로 적재·공간연산, 단가·계수를 DB 기준테이블화 → **작목·지역 확장 시 데이터 추가만으로 즉시 적용**(코드 변경 불필요).

## 10. 기타

- **공공데이터 국외반출 제약 대응**: V-World가 해외 클라우드(배포 서버) IP를 차단함을 확인하고, 국내 회선 기기에 우회 프록시(FastAPI + Tailscale Funnel, 토큰 인증)를 구축해 합법·안정적으로 공간정보를 활용. 미가동 시 정적 폴백으로 데모 무중단.
- **책임 있는 표현**: 모든 화면·리포트에 "인수 검토가 범위(참고용 추정)"와 면책 문구를 명시(법적 감정평가 아님).
- **첨부 예정**: ① 시연 영상(2대 핵심 플로우), ② 포트폴리오(화면 캡처·아키텍처 다이어그램). 양식에서 도표·이미지·영상·별도 파일 첨부가 허용됨.

## 11. 기대효과 *(평가: 주제 시의성 10)*

### 문제의 시급성 (공공통계 근거 — 시장 규모)
- **농가 고령화 역대 최고**: 65세 이상 농가인구 비율 **55.8%**(2024년, 처음 55% 돌파, 전년 대비 +3.2%p). 70세 이상 경영주가 **50.8%(과반)·49만 4,710가구**. *(통계청 2024 농림어업조사)*
- **청년농 붕괴**: 40세 미만 경영주 농가가 2020년 1만 2,426가구 → **2024년 4,601가구**로 4년 만에 약 1/3 토막(역대 최저). *(통계청 2024 농림어업조사)*
- → **수십만 농가가 은퇴를 앞두고 후계자가 없는데, 들어올 청년농과 연결할 통로가 없다.** 이 승계 사각지대가 팜바톤이 푸는 시장.

### 정성적 효과
- **농업 세대 전환·지역소멸 대응**: 후계자 없이 사라지던 영농 기반(농지·과수·시설·판로)을 청년농에게 연결 → 식량안보·농촌 유지에 기여.
- **정보 비대칭 해소**: 고령 농가·청년농 모두에게 공공데이터 기반 객관적 가치·매칭 정보를 무료 제공.

### 정량적 효과(기대)
- 승계 검토 진입장벽: 감정평가 의뢰(비용·수일) → **무료·수십 초**로 단축.
- 매칭 효율: 오프라인 탐색 대신 점수순 추천으로 후보 탐색 시간 대폭 절감.
- 〔보강 권장: 거래액·전환율 등 사업 KPI 추정치는 사업화 단계에서 추가〕

## 12. 제품 및 서비스 개발 실적 *(시제품 완료)*

### 개발 히스토리
- P0: 기준 테이블(시설 표준단가·수령계수) 및 공공데이터 수집·검증
- P1: DB 스키마 + ETL 적재(팜맵 SHP·공시지가·실거래가·소득조사) + 농장 카드 파이프라인
- P2: **결정론적 가치평가 엔진**(테스트 우선 개발) + 농가 등록 플로우 + PDF 리포트
- P3: 매칭 엔진 + 청년농 플로우 + 시드 데이터
- P4: production 배포(Vercel + Railway/Supabase), 인증·역할, 양방향 채팅, V-World 우회 프록시 실가동
- **검증**: pytest 55개 통과, 산식은 `docs/formula.md` 예시 케이스 기반 TDD, production 실동작 확인

### 팀 역량·경력
- 김정훈(대표)·윤채원: **컴퓨터공학 전공** — 풀스택 개발 및 UI/UX 디자인 수행
- 유수민: **농업 계열 전공** — 승계·영농 도메인 지식으로 모델 검증
- 별도 입상 경력 없음(본 대회가 첫 출품). 단, **MVP를 실제 운영 단계까지 완성**한 것이 핵심 실적.
