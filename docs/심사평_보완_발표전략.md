## 5. 사용자·모바일 사용성 검증

### 5-1. 최소 권장 표본

- 고령 농장주: 3~5명
- 청년농 또는 귀농 준비자: 3~5명
- 총 6~10명의 소규모 예비검증

표본이 작다면 반드시 `소규모 예비검증`, `n=○`이라고 표시한다. 충분한 표본처럼 과장하지 않는다.

### 5-2. 농장주 검증 과업

```text
주소 검색
→ 필지 선택
→ 작목·과수 수령 입력
→ 인수 검토 리포트 확인
→ 인수 검토가 범위와 면책 문구 설명
```

확인할 질문:

- 주소와 필지 선택 과정을 스스로 완료할 수 있는가?
- `과수 수령`과 시설 정보 입력 항목을 이해하는가?
- 단일 가격이 아니라 범위로 제공되는 이유를 이해하는가?
- 결과를 감정평가 결과로 오해하지 않는가?
- 다음에 무엇을 준비하거나 확인해야 하는지 알 수 있는가?

### 5-3. 청년농 검증 과업

```text
희망 지역·작목·자본 입력
→ 매칭 리스트 확인
→ 상위 농장 선택
→ 추천 이유와 확인 필요 조건 파악
```

확인할 질문:

- 대표 조건을 혼자 입력할 수 있는가?
- 매칭 점수의 의미를 이해하는가?
- 추천 이유와 확인이 필요한 조건을 구분할 수 있는가?
- 실제 방문·상담 후보를 좁히는 데 도움이 되는가?

### 5-4. 측정 지표

| 지표 | 측정 방법 |
|---|---|
| 과업 완료율 | 도움 없이 마지막 화면까지 완료한 비율 |
| 수행시간 | 시작부터 목표 화면 확인까지 걸린 시간의 중앙값 |
| 도움 요청 | 진행자가 개입한 횟수 |
| 오류·이탈 지점 | 잘못 누르거나 멈춘 화면·입력 항목 |
| 결과 이해도 | 인수 검토가 범위 또는 매칭 이유를 자기 말로 설명하게 함 |
| 사용 편의성 | 5점 척도 또는 간단한 만족도 문항 |

발표 슬라이드에는 지표를 모두 넣지 않고 다음 세 가지 정도만 크게 보여준다.

- 과업 완료율
- 중앙 수행시간
- 결과 이해도 또는 도움 요청 횟수

대표 피드백은 농장주와 청년농 각 1개만 사용한다. 실제 인터뷰 문장을 짧게 요약하고 표본 수를 함께 표시한다.

### 5-5. 검증 결과가 없을 때의 표현

결과를 임의로 만들지 않는다. 다음처럼 기술 검증과 현장 검증을 분리해 말한다.

> 현재 계산의 재현성과 서비스 동작은 테스트로 검증했습니다. 실제 사용 편의성과 현장 정확도는 소규모 사용자 검증 단계이며, 과업 완료율과 결과 이해도를 핵심 지표로 관리하겠습니다.

## 6. 산식의 현장 정확도 검증

### 6-1. 반드시 구분할 개념

- 재현성: 같은 입력에 같은 결과가 나오는가?
- 정확성: 계산 결과가 실제 농장 소득과 가치에 얼마나 가까운가?

결정론적 함수와 pytest는 재현성과 코드 안정성을 뒷받침한다. 이것만으로 현장 정확성이 검증되었다고 말하면 안 된다.

### 6-2. 권장 검증 자료

가능하면 실제 과수농가 3곳 이상의 다음 자료를 확보한다.

- 최근 3년 농업소득 또는 경영기록
- 작목·면적·과수 수령
- 시설 종류·연식·상태
- 실제 거래가격, 전문가 검토가격 또는 비교 가능한 실사자료

### 6-3. 권장 검증 지표

#### 예상 연소득 오차율

```text
|예상 연소득 - 실제 평균소득| ÷ 실제 평균소득 × 100
```

#### 인수 검토가 범위 포함률

```text
실제 거래·검토가격이 팜바톤의 최소~최대 범위 안에 포함된 농장 비율
```

#### 범위 중간값 오차율

```text
|범위 중간값 - 실제 비교가격| ÷ 실제 비교가격 × 100
```

표본이 작을 경우 평균 오차율 하나만 강조하지 말고 농장별 결과를 함께 제시한다.

### 6-4. 자료를 확보하지 못했을 때의 답변

> 현재 검증된 것은 계산의 재현성과 코드 안정성입니다. 실제 농장별 정확도는 충분한 표본을 확보하지 못해 확정적으로 말할 수 없습니다. 앞으로 3년 소득자료와 거래·실사자료를 확보해 예상소득 오차율과 인수 검토가 범위 포함률을 검증하겠습니다.

## 7. 경쟁 비교 슬라이드 전략

### 7-1. 삽입 위치

현재 발표 진행안의 15번 `차별점` 슬라이드를 경쟁 비교 슬라이드로 교체한다.

```text
13. 청년농 매칭·상세 검토
→ 14. 제품·현장 검증
→ 15. 경쟁 비교

```

이 순서는 심사위원의 판단 순서와 맞는다.

1. 실제로 작동하는가?
2. 기존 방식과 무엇이 다른가?
3. 그 차이로 수익을 만들 수 있는가?

### 7-2. 권장 제목

> **농지 탐색과 전문 평가 사이, 승계 사전 검토를 연결합니다**

### 7-3. 비교 대상

특정 경쟁사를 공격적으로 지목하기보다 역할이 다른 서비스 유형으로 비교한다.

1. 농지·매물 플랫폼
2. 감정평가·전문가 의뢰
3. 팜바톤

팜바톤은 감정평가나 공인중개를 대체하지 않는다. 거래 전 후보를 좁히고 승계 논의를 시작할 근거를 만드는 `승계 사전 검토` 서비스로 정의한다.

#### 3) 청년농 매칭 방식

- 지역·작목 검색

#### 4) 이용 목적과 단계

#### 5) 이용 편의성

비용과 소요시간은 실제 측정값 또는 공개 근거가 있을 때만 표시한다.

### 7-6. 정량 비교를 강화할 숫자

실제 확인된 값만 사용한다.

- 농장 구성요소 4개 영역: 농지·작목·시설·판로
- 공공·기준데이터 수
- 매칭 기준 6개
- 매칭 점수 100점
- 리포트 산출시간
- 필수 입력 항목 수
- pytest 55개
- 인수 검토가 최소~최대 범위

`공공데이터 ○종`, `산출 ○초`, `필수 입력 ○개`는 최종 production 서비스에서 직접 세거나 측정한 뒤 확정한다.

### 7-7. 비교 슬라이드에서 피할 표현

- 국내 최초·국내 유일
- 경쟁사 기능을 확인하지 않은 일괄 X 표시
- 근거 없는 비용·소요시간
- 기술 스택 비교
- 주관적인 디자인 우수성
- 감정평가를 대체한다는 표현
- 팜바톤만 모든 항목에서 우수하게 보이는 과도한 표

### 7-8. 발표 멘트

> 기존 서비스와의 차이는 데이터 수가 아니라 판단 범위입니다. 농지·매물 서비스가 토지 탐색을 돕고, 전문가는 공식적인 평가와 거래를 담당합니다. 팜바톤은 그 사이에서 농지와 작목, 시설, 예상소득을 농장 단위로 정리하고 청년농의 6가지 영농 조건과 연결합니다.

### 7-9. 슬라이드 전환 멘트

14번 검증 슬라이드에서 15번 경쟁 비교로:

> 기능이 실제로 동작하는 것과 사용자가 결과를 이해하는 과정까지 검증했습니다. 그렇다면 기존 방식과 비교해 팜바톤은 어떤 역할을 담당하는지 말씀드리겠습니다.

15번 경쟁 비교에서 16번 사업화로:

> 이 차별성을 기반으로 개인에게는 무료로 진입장벽을 낮추고, 초기에는 지자체를 지불 고객으로 확보하겠습니다.

## 8. 시장·수익모델 보완

### 8-1. 수익모델의 핵심 원칙

> 개인은 무료로 모으고, 지자체가 승계 성과에 지불한다.

무료 리포트는 농장 공급을 확보하는 기능이다. 농장주에게 초기 비용을 부과하면 양면 플랫폼의 핵심인 농장 풀이 형성되기 어렵다.

### 8-2. 수익화 순서

| 단계 | 수익모델 | 역할 |
|---|---|---|
| 1단계 | B2G 지자체 구독 | 승계 대상 농가 발굴·청년농 유입·매칭 실적 관리 |
| 2단계 | 제휴 공인중개사 입점·연계 | 거래 단계의 자격자 연결과 반복 매출 |
| 부가 | 실사 연계 정밀 리포트 | 자료·현장 확인이 필요한 단계에서 유료화 |

팜바톤은 무자격 중개보수를 직접 받는다고 표현하지 않는다. 거래는 제휴 공인중개사가 수행하고, 팜바톤은 입점·광고·플랫폼 연계 대가를 받는 구조로 설명한다.

### 8-3. 초기 고객 확보 흐름

```text
농업기술센터·지자체 실증 제안
→ 관내 고령 농가 등록
→ 해당 지역 희망 청년농 유입
→ 상담·매칭 실적 축적
→ 지자체 연간 구독 전환
```

### 8-4. 지자체가 구매하는 이유

지자체는 단순한 대시보드 화면을 구매하는 것이 아니다.

- 승계 위험 농가 발굴
- 청년농 유치 대상자 파악
- 등록·매칭·상담 실적 집계
- 분기·연간 성과보고
- 기존 지원사업 노출과 참여 추적

발표 핵심 문장:

> 지자체는 대시보드가 아니라 정책 대상자 발굴과 청년농 유치 실적을 구매합니다.

### 8-5. 매출 산식 제시 방법

숫자를 나열하지 말고 다음 곱셈식으로 보여준다.

```text
대상 시군 수 × 유료 전환율 × 평균 연간 구독료 = 연간 B2G 매출
```

현재 내부 사업화 문서의 예시 시나리오:

```text
3개 도 약 48개 시군
× 3년차 전환율 20%
× 평균 연 900만 원
= 약 10개 지자체, 연 9,000만 원
```

이 수치는 확정 매출이 아니라 `사업화 가정 시나리오`라고 표시한다. 가격·전환율은 지자체 인터뷰와 실증 협의를 통해 검증해야 한다.

### 8-6. 발표 전 확보하면 좋은 사업성 증거

최소 3개 지자체·농업기술센터에 접촉하고 다음 중 하나를 확보한다.

- 담당자 인터뷰
- 후속 미팅 일정
- 실증 검토 의사
- 도입 시 필요한 기능
- 가능한 예산 항목에 대한 의견
- 농장주 또는 청년농 모집 협력 가능성

발표 표현:

> ○개 기관에 제안했고, ○개 기관 담당자와 실증 범위와 구매 조건을 논의했습니다.

`관심을 보였다`처럼 모호한 표현보다 접촉 수, 미팅 수, 논의 주제를 구체적으로 제시한다.

## 11. 예상 질문과 답변

### Q1. 실제 사용자가 써봤는가?

검증 결과가 있을 때:

> 농장주 ○명과 청년농 ○명을 대상으로 모바일 과업 테스트를 진행했고, 완료율과 수행시간, 리포트 이해도를 측정했습니다.

아직 충분하지 않을 때:

> 아직 충분한 베타 표본은 없습니다. 기술 검증과 현장 검증을 구분하고 있으며, 현재 ○명을 대상으로 소규모 예비 사용성 검증을 진행하고 있습니다.

### Q2. 같은 결과가 나온다고 정확한 것은 아니지 않은가?

> 맞습니다. 재현성과 정확도는 다릅니다. 재현성은 결정론 산식과 테스트로 검증했고, 정확도는 실제 소득·거래·실사자료와 비교한 오차율과 범위 포함률로 별도 검증하겠습니다.

### Q3. 누가 돈을 내는가?

> 초기 지불 고객은 개인이 아니라 지자체입니다. 개인은 무료로 유지해 농장과 청년농 풀을 확보하고, 지자체에는 승계 대상 발굴과 매칭 실적 관리 기능을 연간 구독으로 제공합니다.

### Q4. 지자체가 왜 구매하는가?

> 지자체는 대시보드 화면이 아니라 정책 대상 농가 발굴과 청년농 유치 실적을 구매합니다.

### Q5. 기존 농지 서비스와 무엇이 다른가?

> 기존 서비스가 농지나 매물을 보여준다면, 팜바톤은 작목·시설·예상소득까지 농장 단위로 구조화하고 청년농의 영농계획과 연결합니다.

### Q6. 감정평가를 대체하는가?

> 대체하지 않습니다. 팜바톤의 결과는 감정가가 아니라 승계 논의를 시작하기 위한 인수 검토가 범위입니다. 공식 평가나 거래가 필요한 단계에서는 전문가와 연결됩니다.

### Q7. 중개수수료를 받을 수 있는가?

> 팜바톤이 중개보수를 직접 받는 구조가 아닙니다. 거래는 제휴 공인중개사가 담당하고, 팜바톤은 입점·광고·플랫폼 연계 대가를 받는 구조입니다.

### Q8. 초기 농장과 청년농은 어떻게 확보하는가?

> 공급 측은 농업기술센터와 지자체를 통해 승계 의향 농가를 모집하고, 수요 측은 청년농 교육과정과 귀농·귀촌 지원사업 참여자를 지역·작목 조건으로 유입시키겠습니다.

### Q9. 시장이 너무 좁지 않은가?

> 충북·경북·충남의 과수 3종은 현장 검증을 위한 시작점입니다. 팜바톤의 차별성이 토지 외 자산에 있으므로, 이후에는 과수 전체와 시설원예처럼 작목과 시설 가치가 큰 영역부터 확장하겠습니다.
